# Заполнение эмбеддингов для векторного поиска (Yandex Cloud)

В таблице `cars` есть колонка `embedding` (тип `vector(256)`). Она заполняется через **Yandex Cloud Foundation Models API** (модель эмбеддингов `text-search-doc`). После заполнения можно реализовать поиск по смыслу (семантический поиск).

---

## 1. Что нужно в Yandex Cloud

1. **Каталог (folder)** в [Yandex Cloud](https://console.cloud.yandex.ru/).
2. **Сервисный аккаунт** с ролью **`ai.languageModels.user`** (доступ к моделям YandexGPT/эмбеддингам).
3. **API-ключ** этого сервисного аккаунта.

### Где взять Folder ID (ID каталога)

ID каталога — это строка вида `b1g7gvsi89m3...` (начинается с `b1g`). Получить можно так:

**Способ 1: из адресной строки браузера**

1. Откройте [консоль Yandex Cloud](https://console.cloud.yandex.ru/).
2. В левом верхнем углу нажмите на **название текущего каталога** (или «Выбрать каталог»).
3. Выберите нужный каталог из списка.
4. Посмотрите в **адресную строку браузера**: после перехода часто в URL есть параметр `folderId=...` или путь вида `.../folders/b1g...` — это и есть Folder ID. Скопируйте значение после `folderId=` или сегмент с `b1g...`.

**Способ 2: страница «Облака и каталоги»**

1. В консоли в меню слева найдите раздел **«Управление»** (или **«Billing»** / «Биллинг»).
2. Перейдите в **«Облака и каталоги»** (или **«Cloud and folders»**).
3. В списке каталогов нажмите на **название** нужного каталога.
4. Откроется страница каталога — в блоке **«Общая информация»** или **«Информация о каталоге»** должно быть поле **«ID»** или **«Идентификатор каталога»**. Скопируйте это значение.

**Способ 3: через CLI Yandex Cloud**

Если установлен [YC CLI](https://cloud.yandex.ru/docs/cli/quickstart), выполните в терминале:

```bash
yc resource-manager folder list
```

В колонке `ID` будет указан идентификатор каждого каталога.

---

### Сервисный аккаунт и API-ключ

1. В консоли откройте раздел **«Сервисные аккаунты»** (меню слева: IAM или «Управление доступом»).
2. Создайте сервисный аккаунт и назначьте ему роль **«AI Language Models User»** (`ai.languageModels.user`) на нужный каталог.
3. Откройте карточку этого аккаунта → **«Создать API-ключ»**. Заполните описание (по желанию) и нажмите «Создать». Сохраните выданный ключ — он показывается один раз. Отдельного поля «область действия» в форме нет: доступ к API задаётся ролью аккаунта на каталог (шаг 2).

**Официальная инструкция:** [Управление API-ключами — Yandex Cloud](https://yandex.cloud/ru/docs/iam/operations/authentication/manage-api-keys#console_1).

---

### Если скрипт возвращает 403 (Permission denied)

Ошибка значит: у сервисного аккаунта нет прав на каталог. Сделайте по шагам:

1. **Откройте нужный каталог**  
   В [консоли](https://console.cloud.yandex.ru/) в левом верхнем углу выберите каталог, ID которого указан в `YANDEX_FOLDER_ID` в `.env`.

2. **Перейдите в «Управление доступом»**  
   В меню слева: **«Управление»** → **«Управление доступом»** (или **IAM**). Убедитесь, что вы находитесь в контексте **этого каталога**, а не облака или организации.

3. **Найдите сервисный аккаунт**  
   В списке участников найдите сервисный аккаунт, для которого вы создавали API-ключ (тот, что в `YANDEX_API_KEY`).

4. **Проверьте роли**  
   У этого аккаунта должна быть роль **«Пользователь моделей машинного обучения»** (или **AI Language Models User** / `ai.languageModels.user`). Если роли нет — нажмите **«Назначить роль»** → выберите этот сервисный аккаунт → роль **«Пользователь моделей машинного обучения»** → субъект: каталог (текущий). Сохраните.

5. **Подождите 1–2 минуты** и снова запустите:  
   `python scripts/populate_cars_embeddings.py --limit 5`

Если у вас организация (organization): роль может потребоваться и на уровне организации, или сервисный аккаунт и ключ должны быть в том же каталоге/облаке, где включён биллинг и доступ к YandexGPT/Foundation Models.

## 2. Настройка окружения

В каталоге `carmatch-backend` в файле **`.env`** добавьте (или отредактируйте):

```env
# Эмбеддинги Яндекса
YANDEX_FOLDER_ID=b1gxxxxxxxxxxxxxxxxxx
YANDEX_API_KEY=AQVNxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

- `YANDEX_FOLDER_ID` — ID каталога (строка вида `b1g...`).
- `YANDEX_API_KEY` — API-ключ сервисного аккаунта.

Также должен быть настроен **`DATABASE_URL`** для PostgreSQL с расширением pgvector (как в основном README проекта).

---

## 3. Запуск скрипта заполнения

Из корня **carmatch-backend** (из каталога, где лежат `src/` и `scripts/`):

```bash
# Обработать все машины без эмбеддинга (по умолчанию)
python scripts/populate_cars_embeddings.py

# Ограничить число записей (например, для проверки)
python scripts/populate_cars_embeddings.py --limit 100

# Перезаписать эмбеддинги у всех активных cars
python scripts/populate_cars_embeddings.py --force

# Увеличить паузу между запросами к API (по умолчанию 0.2 с)
python scripts/populate_cars_embeddings.py --delay 0.5
```

Скрипт:

- Берёт из БД активные записи `cars` (`is_active = true`).
- Без `--force` обрабатывает только те, у которых `embedding` ещё `NULL`.
- Для каждой машины формирует текст: марка, модель, модификация, тип кузова, год, описание (как в `format_car_description`).
- Отправляет текст в Yandex Embeddings API и получает вектор длины 256.
- Записывает вектор в `cars.embedding`.

---

## 4. Проверка

После запуска можно проверить в БД:

```sql
SELECT id, mark_name, model_name, embedding IS NOT NULL AS has_embedding
FROM cars
WHERE is_active = true
LIMIT 10;
```

Или через psql в контейнере:

```bash
docker exec carmatch-postgres psql -U carmatch -d carmatch -c "SELECT id, mark_name, embedding IS NOT NULL AS has_emb FROM cars LIMIT 5;"
```

## 5. Ограничения и советы

- **Лимиты API**: у Yandex Cloud есть [квоты на запросы](https://cloud.yandex.ru/docs/foundation-models/concepts/limits). При большом числе машин используйте `--delay` или разбейте запуск на несколько раз с `--limit`.
- **Размерность**: модель `text-search-doc` возвращает вектор длины **256**. Колонка в БД уже имеет тип `vector(256)` — менять ничего не нужно.
- **Текст для эмбеддинга**: берётся то же представление, что и для отображения пользователю (марка, модель, модификация, кузов, год, текстовое описание). При изменении логики формирования описания имеет смысл перезаполнить эмбеддинги с `--force`.
- **Без интернета**: скрипт обращается к `llm.api.cloud.yandex.net` — нужен доступ в интернет.

---

## 6. Дальнейшие шаги

После заполнения `cars.embedding` можно:

- Реализовать эндпоинт векторного поиска: по тексту запроса получить эмбеддинг (модель `text-search-query`), затем искать в БД по косинусному расстоянию/близости (pgvector: `<=>`, `ORDER BY embedding <=> query_embedding`).
- Добавить индексы pgvector (например, IVFFlat) при большом количестве записей для ускорения поиска.

Таблицу **cars.xml** скрипт не изменяет — работа идёт только с таблицей **cars** в базе данных.
